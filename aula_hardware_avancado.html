MOD_01

Fundamentos Avançados

Antes de mergulhar nas ferramentas, é essencial entender o que acontece internamente em cada subsistema. Hardware avançado cobre quatro grandes domínios de análise:

🧠

CPU & Cache

Registradores, pipeline, hierarquia de cache L1/L2/L3, hyperthreading, boost clocks e TDP real vs nominal.

  • Instruções por ciclo (IPC)
  • Branch prediction & speculative execution
  • Thermal throttling e power limits (PL1/PL2)
  • Microarquitetura (Zen 4, Raptor Lake)
🗄️

Memória RAM

Timings CAS, sub-timings, dual-rank vs single-rank, JEDEC vs XMP/EXPO e latência real medida.

  • tCL, tRCD, tRP, tRAS primários
  • Frequência vs latência absoluta
  • Topologia daisy-chain vs T-topology
  • IMC (Integrated Memory Controller)
💽

Armazenamento

NVMe PCIe Gen4/5, SATA AHCI, QD (Queue Depth), latência de acesso aleatório 4K e endurance TBW.

  • Sequential vs Random I/O
  • SLC/MLC/TLC/QLC NAND
  • Wear leveling & TRIM
  • HMB (Host Memory Buffer)
🌐

Rede & Barramentos

PCIe lanes, largura de banda, USB 3.2/4.0, Thunderbolt 4 e análise de gargalos de I/O.

  • PCIe 3.0 vs 4.0 vs 5.0 bandwidth
  • Bifurcação e switching de lanes
  • Latência de rede (jitter, packet loss)
  • Chipset vs CPU lanes diretas
⚡

Energia & Térmica

VRM (Voltage Regulator Module), fases de energia, eficiência 80 PLUS, temperatura de junção vs case.

  • Ripple de voltagem
  • TjMax e Tcase
  • Curva P-State / C-State
  • PSU wattage real vs pico
🎮

GPU Avançado

VRAM bandwidth, shader occupancy, PCIe bandwidth vs VRAM bandwidth, core clocks e memory clocks.

  • ROPs, TMUs, CUDA/Compute Units
  • Power limit e voltage curve
  • GDDR6 vs GDDR6X vs HBM3
  • PCIe riser degradation
⚠️

Atenção: Monitoramento de hardware em baixo nível pode requerer desabilitar o Secure Boot e habilitar acesso a drivers não assinados. Faça sempre em ambiente controlado quando testar novas ferramentas.

MOD_03

CPU & Memória: Como Usar

HWiNFO64 — Configuração Essencial

O HWiNFO64 é a ferramenta mais completa para monitoramento em tempo real. Siga este fluxo:

01

Abrir em Modo Sensores

Ao abrir, marque Sensors Only para ir direto ao painel. Desmarque Show Summary Window para a janela principal compacta. Isso evita a tela de resumo e vai direto aos dados.

02

Configurar Sensores Relevantes

Clique com botão direito em qualquer sensor → Add to Tray para monitorar na bandeja. Essenciais: CPU (Tctl/Tdie) temperatura real, CPU Package Power, DRAM Voltage, VCore, CPU Clock.

03

Logar Dados com CSV

Settings → Start Logging on Startup → intervalo de 1000ms. Gera CSV para análise em Excel ou Python. Essencial para caçar throttling intermitente que não aparece em monitoramento manual.

04

Interpretar CPU Power Limits

Observe CPU Package Power vs PL1 e PL2. Se Package Power fica travado exatamente no PL1 por mais de 56 segundos, o sistema está limitado por energia — não por temperatura. Procure "Power Limit Throttling = Yes" nos sensores.

CPU-Z — Lendo Timings de Memória

Aba SPD

  • Slot selecionado (compare os dois pentes)
  • Max bandwidth nominal JEDEC
  • Perfis XMP/EXPO disponíveis e timings
  • Fabricante real do chip (Hynix/Samsung/Micron)
  • Date code: semana e ano de fabricação

Aba Memory

  • Frequência real rodando no momento
  • Timings primários: tCL-tRCD-tRP-tRAS
  • Command Rate (1T ou 2T)
  • DRAM Frequency (metade da DDR efetiva)
  • Dual Channel confirmado ou não

Comparativo: Latência Absoluta vs Frequência

CrystalDiskMark — Interpretando os Resultados

NVMe PCIe Gen4 Típico 
──────────────────────────────────────────────────
SEQ1M  Q8T1   Read:  7.100 MB/s  Write: 6.600 MB/s
SEQ1M  Q1T1   Read:  5.800 MB/s  Write: 5.200 MB/s
RND4K Q32T16  Read:    850 MB/s  Write:   710 MB/s
RND4K  Q1T1   Read:     88 MB/s  Write:    74 MB/s
──────────────────────────────────────────────────
  ↑ RND4K Q1T1 = uso real do dia a dia (mais importante)

SEQ (Sequencial)

  • Transferência de arquivos grandes (backups, vídeos)
  • Q8T1 = Queue Depth 8, Thread 1
  • Número anunciado na caixa pelo fabricante
  • Fácil de saturar com SLC write cache
  • Teste real: arquivo > 50GB para forçar saída do cache

RND 4K (Aleatório)

  • Operações do OS, apps, banco de dados
  • Q1T1 = uso real single-thread do dia a dia
  • Diferencia SSD barato de premium na prática
  • HDD: ~0.5 MB/s | SATA: ~50 MB/s | NVMe: ~88-120 MB/s
  • Latência NVMe: ~100µs vs HDD: ~10ms (100x mais rápido)
MOD_05

Rede & Barramentos

Análise de NIC e Controlador de Rede

01

Identificar o Chipset da NIC

Device Manager → Network Adapters → propriedades. O chipset real (Intel I225-V, Realtek 8125B) determina latência e CPU offloading. Intel NICs têm melhor latência e suportam RSS (Receive Side Scaling) para distribuir carga por múltiplos cores.

02

Testar Throughput Real com iperf3

Execute servidor na outra máquina: iperf3 -s, e no cliente:

iperf3 -c 192.168.1.10 -t 30 -P 4 --bidir
# -P 4 = 4 streams paralelos (simula tráfego real)
# --bidir = upload e download simultâneos
# Resultado Gigabit esperado: ~940 Mbits/sec
03

Wireshark — Filtros Essenciais

Para hardware analysis: tcp.analysis.retransmission para instabilidade de rede; icmp para latência detalhada; frame.len < 64 para runt frames indicando problema físico (cabo, NIC com mau contato).

PCIe — Verificar Largura e Velocidade Real

Windows PowerShell / Device Manager 
# No GPU-Z: clique em "?" ao lado de Bus Interface
# para forçar um teste de link width ativo

GPU:   PCIe 4.0 x16 = 32 GB/s bidirecional
NVMe:  PCIe 4.0 x4  =  8 GB/s bidirecional
PCIe 3.0 x16:         = 16 GB/s bidirecional

# "x8 @ x16" no GPU-Z = slot é x16 mas só x8 elétrico
# Perda de 5-8% (high-end), até 25% (RTX 4090, RX 7900)
ℹ️

Dica: Muitas placas B-series reduzem o slot primário para x8 quando um segundo SSD NVMe é instalado no slot M.2 secundário. Verifique o manual da placa-mãe para as restrições de bifurcação de lanes.

MOD_06

Stress Test & Diagnóstico

Protocolo de Stress Test — Ordem Correta

Teste cada componente em isolamento antes de combinar:

01

RAM Primeiro — MemTest86 (2 passes)

RAM com erro causa crashes em qualquer stress test subsequente. Grave em USB bootável, reinicie e aguarde 2 passes completos (~4h para 32GB). Qualquer erro vermelho = problema real de memória.

02

CPU Isolado — Prime95 SmallFFTs (30min)

SmallFFTs estressam maximamente o núcleo sem afetar RAM. Use com HWiNFO64 aberto monitorando temperatura e power. Se temperatura superar 95°C Intel ou 90°C AMD Tdie, pare e revise pasta térmica e cooler.

# No Prime95: Options → Torture Test
Small FFTs   → Máximo calor CPU, mínimo RAM
Large FFTs   → Estressam L3 cache e RAM também
Blend        → Mix CPU + RAM (use por último)
03

GPU — FurMark (15-20min) + Afterburner

Monitore: GPU Temp (max 85°C), VRAM Temp (max 95°C para GDDR6X), GPU Power (deve estabilizar no TDP configurado). Se temp subir além do limite, verifique pasta térmica da GPU e fluxo de ar do case.

04

AIDA64 Full System — 1 Hora

Com os três anteriores OK, rode AIDA64 com CPU + FPU + Cache + Memory + GPU por 1 hora. Revela problemas de PSU (quedas de voltagem), VRM superaquecendo, e pasta térmica inadequada em carga combinada real.

MOD_07

BIOS & Configurações

⚙️

XMP / EXPO

Habilitar libera o perfil otimizado da RAM. Procure em AI Tweaker / OC / Extreme Tweaker dependendo do fabricante.

  • XMP 3.0: suportado por Intel Alder Lake+
  • EXPO: padrão AMD para Ryzen 7000+
  • Se instável: tente o segundo perfil XMP
  • Confirmar com CPU-Z aba Memory após reboot
🔋

Power Limits

PL1 = limite sustentado (long-term). PL2 = boost de curto prazo. Muitos OEMs deixam ambos no máximo, causando throttling.

  • Intel: "Long/Short Duration Power Limit"
  • AMD: PPT/TDC/EDC configuráveis
  • Reduzir PL1 melhora thermals sem perder muito
  • PBO (AMD) para boost inteligente automático
🌀

Fan Curves (PWM)

Curvas de fan no BIOS permitem controle preciso de temperatura vs ruído. Use modo PWM para controle granular.

  • 40°C → 30% PWM (silencioso)
  • 80°C → 80% PWM (carga normal)
  • 90°C → 100% PWM (emergência)
  • Zero RPM mode abaixo de 40°C
🔬

Resizable BAR / SAM

Permite à CPU acessar toda VRAM da GPU. Ganho de 5–15% em jogos modernos sem custo.

  • Requer: "Above 4G Decoding" habilitado
  • Intel: padrão em Z690+ (Alder Lake)
  • AMD SAM: B550/X570 com BIOS atualizado
  • Verificar em GPU-Z: "Resizable BAR: Yes"
🛡️

Secure Boot & TPM

Secure Boot impede drivers não assinados. Necessário desabilitar para algumas ferramentas de diagnóstico de baixo nível.

  • MemTest86: funciona com Secure Boot ON
  • Alguns drivers de OC: requerem desabilitar
  • TPM 2.0: necessário para Windows 11
  • Reabilitar sempre após testes concluídos
📡

PCIe Gen Lock

Forçar PCIe Gen específico garante que dispositivos operem na velocidade correta e evita auto-negotiation bugs.

  • Slot M.2: Gen3/4/5 — verifique suporte do SSD
  • Slot GPU: sempre maximize (Gen4 ou Gen5)
  • Gen3 Force: para SSDs antigos com bugs Gen4
  • Confirmar velocidade ativa no GPU-Z/HWiNFO

10 Exercícios Práticos

Do iniciante ao avançado — cada exercício foi desenhado para construir habilidade real com as ferramentas aprendidas. Complete-os na ordem para melhor progressão.

01
● Iniciante

Mapeamento Completo do Hardware

Documente cada componente do seu sistema com as informações de baixo nível que as ferramentas expõem.

HWiNFO64CPU-ZGPU-Z
Objetivo

Criar uma tabela com: CPU stepping/revision/TDP real, RAM fabricante do chip/timings ativos/dual channel, GPU vbios/subvendor/compute units, SSD controlador/tipo NAND/firmware.

Dica de execução

No CPU-Z aba "CPU" anote "Stepping" e "Revision". Na aba "SPD" alterne entre slots para comparar os dois pentes. No GPU-Z clique no ícone de pasta para ver histórico de versões de VBIOS.

02
● Iniciante

Auditoria de Saúde dos Discos

Análise completa dos dados SMART e estimativa de vida útil restante de todos os discos do sistema.

CrystalDiskInfoCrystalDiskMark
Objetivo

Para cada disco: registrar atributos SMART críticos (05, C5, C6, 09, C2, E9), rodar benchmark com 5 passes de 1GiB, comparar resultado com spec sheet do fabricante e classificar saúde geral com justificativa.

Dica de execução

No CrystalDiskMark, mude o tamanho de teste para 16GiB. Isso força o SSD a sair do SLC write cache e mostra a velocidade real da NAND TLC/QLC. A queda de velocidade entre 1GiB e 16GiB revela a capacidade do SLC cache.

03
● Iniciante

Baseline de Temperatura em Idle

Estabelecer linha de base de temperatura de todos os componentes em idle verdadeiro, sem processos em background.

HWiNFO64Task Manager
Objetivo

Fechar todos os apps, aguardar 10 minutos, ativar logging no HWiNFO64 por 5 minutos. Registrar média e máximo de: CPU temp, GPU temp, SSD temp, VRM temp e CPU Package Power em idle verdadeiro.

Dica de execução

Ordene o Task Manager por CPU e feche qualquer processo consumindo acima de 1%. Desabilite temporariamente antivírus e Windows Update para idle verdadeiro. Esse baseline será sua referência para comparar futuros testes.

04
◆ Intermediário

Comparativo XMP vs JEDEC

Medir e comparar latência absoluta, bandwidth e performance real com e sem perfil XMP/EXPO ativo.

CPU-ZAIDA64Cinebench R23
Objetivo

Rodar AIDA64 Memory Benchmark e Cinebench nT em dois boots: JEDEC padrão e XMP/EXPO ativado. Calcular diferença percentual em Read/Write/Copy bandwidth e Memory Latency. Documentar os timings ativos em cada caso via CPU-Z.

Dica de execução

No AIDA64: Tools → Memory Benchmark. O campo "Memory Latency" em nanossegundos é onde XMP mais impacta workloads sensíveis. Para Ryzen 7000, XMP 6000MHz CL30 costuma dar a melhor relação frequência/IF (Infinity Fabric) 1:1.

05
◆ Intermediário

Stress Test Isolado de CPU

Validar estabilidade térmica e elétrica da CPU sob carga máxima, identificando throttling por temperatura ou por power limits.

Prime95HWiNFO64
Objetivo

Rodar SmallFFTs por 30 minutos com log HWiNFO64. Abrir o CSV e criar gráfico de: temperatura, Package Power e Clock Speed. Identificar se houve throttling, classificar o tipo (thermal ou power limit) e propor solução.

Dica de execução

Throttling térmico: clock cai E temperatura está no TjMax. Throttling por power limit: clock cai mas temperatura está ~70°C e "Power Limit Throttling = YES" no HWiNFO64. São causas distintas com soluções distintas.

06
◆ Intermediário

Benchmark de GPU e Análise PCIe

Medir performance de GPU e verificar se o link PCIe está operando na largura e velocidade corretas.

3DMarkGPU-ZMSI Afterburner
Objetivo

Rodar Time Spy e Fire Strike. No GPU-Z verificar Bus Interface durante o teste (deve mostrar PCIe x16). Comparar score com média global no site do 3DMark. Ativar overlay do Afterburner monitorando GPU Temp, Power, Core Clock e VRAM Usage.

Dica de execução

No GPU-Z, clique no "?" ao lado de Bus Interface para forçar um link test. O valor "x8 @ x16" significa que o slot é fisicamente x16 mas elétrico x8 — comum em placas B-series mid-range. Impacto depende da GPU: irrelevante em RTX 3060, significativo em RTX 4090.

07
◆ Intermediário

Análise de Throughput de Rede Local

Medir throughput real entre dois dispositivos e identificar gargalos de hardware ou software na stack de rede.

iperf3WiresharkHWiNFO64
Objetivo

Medir: TCP single stream, TCP multi-stream (4 threads), UDP com target 900Mbps. Capturar com Wireshark e medir CPU usage da NIC. Comparar Wi-Fi 6 vs Gigabit Ethernet e documentar jitter e packet loss em cada cenário.

Dica de execução

Servidor: iperf3 -s | TCP: iperf3 -c [IP] -P 4 -t 20 | UDP: iperf3 -c [IP] -u -b 900M -t 20. No resultado UDP, jitter acima de 5ms indica problema. No Wireshark, filtre tcp.analysis.retransmission para ver retransmissões.

■ Avançado

Stress Test Full System — Protocolo 1h

Validação completa de estabilidade do sistema sob carga máxima simultânea de todos os subsistemas com análise de dados.

AIDA64HWiNFO64CrystalDiskMark
Objetivo

Rodar AIDA64 System Stability (CPU+FPU+Cache+Memory+GPU) por 60 minutos com log contínuo. Executar CrystalDiskMark em loop simultâneo. Analisar CSV e criar relatório identificando picos térmicos, quedas de clock e variações de voltagem com correlações temporais.

Dica de execução

No Excel, crie gráficos de linha sobrepostos para temp CPU, temp GPU, CPU Core Voltage e CPU Clock com o mesmo eixo de tempo. Quedas de clock correlacionadas com temperatura = thermal throttle. Quedas com temp OK = power limit ou VRM throttle (verifique VRM temp também).

09
■ Avançado

Otimização Manual de Timings RAM

Ajuste manual de sub-timings de RAM para maximizar latência e bandwidth mantendo estabilidade total comprovada.

BIOSMemTest86AIDA64CPU-Z
Objetivo

Partindo do perfil XMP: reduzir tCL em 2 passos, validar com MemTest86 (1 pass), medir latência com AIDA64. Repetir para tRCD e tRP. Documentar: cada configuração testada, resultado do MemTest e latência medida em cada etapa.

Dica de execução

Mude UM timing por vez e valide antes de continuar. Se MemTest86 detectar erros, aumente 1 step e valide novamente. Kits DDR5 frequentemente toleram tCL28-30 mesmo vendidos como CL36 — o potencial depende do die (Samsung B-die, Hynix M-die, etc.).

10
■ Avançado

Diagnóstico de Instabilidade Intermitente

Identificar a causa raiz de crashes esporádicos por meio de evidências múltiplas e cruzamento de dados de hardware.

HWiNFO64MemTest86Event ViewerPrime95CrystalDiskInfo
Objetivo

Montar relatório completo: (1) checar Event Viewer por IDs críticos (41, 129, 7), (2) verificar SMART para bad sectors, (3) MemTest86 completo 2 passes, (4) analisar log HWiNFO64 anterior ao crash, (5) Prime95 Blend 2h. Concluir com hipótese principal sustentada por evidências múltiplas cruzadas.

Dica de execução

Event Viewer: Windows Logs → System → filtrar Critical+Error. ID 41 = crash inesperado (kernel power), ID 129 = reset de controlador de disco, ID 7 = bad block. Combine com SMART: se ID 41 coincide com SSD temperature acima de 80°C nos logs do HWiNFO64, o disco superaquecendo é o suspeito principal.
